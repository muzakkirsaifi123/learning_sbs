# MuzOps Learning Zone

## Purpose

This is a personal learning knowledge base for DevOps, Kubernetes, Kafka, and Cloud Engineering.

Notes are written in **Notion** and automatically synced to this repo as Markdown files. The site is built with **MkDocs Material** and published to **GitHub Pages**. Every page here is something worked through hands-on — no copy-paste theory.

What's inside:
- **Kubernetes** — operators, controllers, CRDs, reconcile loops
- **Kafka** — architecture, schema registry, cp-schema internals
- **Notion Sync** — pages pulled from Notion and auto-published here

## Local Setup

### 1. Clone the repo

```bash
git clone https://github.com/saifi/learning_sbs
cd learning_sbs
```

### 2. Install dependencies

```bash
pip install mkdocs-material mkdocs-awesome-pages-plugin notion-client
```

### 3. Create your `.env` file

Copy the example and fill in your values:

```bash
cp .env.example .env
```

Edit `.env`:

```
NOTION_TOKEN=ntn_xxxxxxxxxxxxxxxxxxxx
NOTION_PAGE_ID=1fd9f2a93248826a83de816c57fc66b8
```

**How to get these values:**
- `NOTION_TOKEN` — go to [notion.so/profile/integrations](https://www.notion.so/profile/integrations), create an Internal integration, copy the token
- `NOTION_PAGE_ID` — open your Notion page, copy the last part of the URL: `notion.so/Page-Title-**<this-part>**`
- Share your Notion page with the integration: open the page → `...` → Connections → select your integration

## Run Locally to Validate

### Sync pages from Notion

```bash
NOTION_TOKEN="ntn_xxx..." NOTION_PAGE_ID="1fd9f2a9..." python3 notion_sync.py
```

This will:
- Pull all pages from your Notion root page
- Save them as `.md` files under `docs/notion/`
- Auto-update the `nav` section in `mkdocs.yml`
- Regenerate `docs/index.md` with quick links

### Preview the site in browser

```bash
mkdocs serve
```

Open [http://127.0.0.1:8000](http://127.0.0.1:8000) — live reloads on every file change.

### Validate the build (strict mode)

```bash
mkdocs build --strict
```

`--strict` treats warnings as errors — same behaviour as the CI pipeline.
If this passes locally, the pipeline will pass too.

## Run from Pipeline (GitHub Actions)

### Step 1 — Add GitHub Secrets

Go to your repo → **Settings → Secrets and variables → Actions → New repository secret**

Add both:

| Secret name | Value |
|-------------|-------|
| `NOTION_TOKEN` | Your `ntn_xxx...` token |
| `NOTION_PAGE_ID` | Your Notion page ID |

### Step 2 — Enable GitHub Pages (one-time)

Go to repo → **Settings → Pages**

- Source: `Deploy from a branch`
- Branch: `gh-pages` / `/ (root)`
- Save

Your site will be live at `https://<username>.github.io/<repo>/`

### Step 3 — Trigger workflows

**Sync Notion pages manually:**
1. Go to repo → **Actions** tab
2. Select **Sync Notion to MkDocs**
3. Click **Run workflow → Run workflow**

**Deploy site manually:**
1. Go to **Actions → Deploy MkDocs → Run workflow**

Or simply push any change to `main` — deploy triggers automatically.

### Automated schedule

| Workflow | Runs automatically |
|----------|--------------------|
| `notion_sync.yml` | Every 6 hours — syncs Notion → commits to repo → triggers deploy |
| `deploy.yml` | On every push to `main` — builds and publishes to GitHub Pages |

Full automated flow:

```
Write notes in Notion
      ↓  every 6 hours  (notion_sync.yml)
docs/notion/*.md saved and committed to main
      ↓  on push to main  (deploy.yml)
MkDocs site built and published to GitHub Pages
```
